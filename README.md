# Security Awareness Training Program: Implementation Blueprint

Welcome to the Security Awareness Training Program. This comprehensive, practical, and measurable program is designed specifically for organizations looking to reduce their vulnerability to social engineering and password-related cyber threats. 

This program is structured to be delivered by non-technical HR, Compliance, or IT managers. It includes training modules, simulation guidelines, baseline/post assessments, and an effectiveness evaluation framework.

## 2. 8-Week Rollout Timeline

Deploy the program using this chronological sequence to maximize retention and keep data clean.

```
Week 1: Prep & Whitelisting 
Week 2: Baseline Pre-Assessment 
Week 3: Module 1 & 2 Release
Week 4: Module 3 & Live Sessions (Facilitated)
Week 5: Phishing Simulation (Easy)
Week 6: Social Eng. Drills (Vishing/Audits)
Week 7: Post-Training Assessment
Week 8: KPI Compilation, Reporting & Review
```

### Week 1: Program Setup & Alignment
*   IT coordinates with the Compliance Manager to whitelist simulation email domains and set up reporting buttons.
*   The program coordinator prepares physical handouts and schedules live session slots.

### Week 2: Baseline Knowledge Measurement
*   Send out the [pre-training-assessment.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/assessments/pre-training-assessment.md) link to all employees.
*   Compile initial score data and establish your organizational baseline using the [scoring-rubric.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/assessments/scoring-rubric.md).

### Week 3: Phishing & Password Training Rollout
*   Distribute Module 1 (Phishing) and Module 2 (Password Hygiene) via your LMS or email.
*   Encourage employees to complete the self-assessments and set up password managers.

### Week 4: Social Engineering & Live Facilitated Sessions
*   Distribute Module 3 (Social Engineering).
*   Conduct 45-minute live review sessions for teams using the [facilitator-guide.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/training-modules/facilitator-guide.md) to practice link inspection and physical security drills.

### Week 5: Phishing Simulation Campaign
*   Execute the first benign phishing drill using Template 1 (Urgent Password Expiration) from the [phishing-simulation-plan.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/simulations/phishing-simulation-plan.md).
*   Log click rates, open rates, and report rates. Redirect clickers to the educational landing page.

### Week 6: Social Engineering & Vishing Audits
*   Conduct in-person desk checks and phone-based vishing scenarios using instructions from [social-engineering-simulation.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/simulations/social-engineering-simulation.md).
*   Record passes/failures by department.

### Week 7: Post-Training Assessment
*   Send the [post-training-assessment.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/assessments/post-training-assessment.md) link to all employees.
*   Calculate the average score improvement (Delta) compared to Week 2.

### Week 8: Executive Reporting & Iteration
*   Present program KPIs (Completion, Click Rate, Report Rate, MTTR) to executive management using the [reporting-template.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/effectiveness-framework/reporting-template.md).
*   Initiate the [improvement-plan.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/effectiveness-framework/improvement-plan.md) remediation steps for low-scoring departments or repeat clickers.

## 3. Quick-Start Guide for Program Coordinators

1.  **Read the Guides:** Familiarize yourself with the [facilitator-guide.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/training-modules/facilitator-guide.md) and [scoring-rubric.md](file:///c:/Users/DELL/OneDrive/Desktop/awareness/assessments/scoring-rubric.md).
2.  **Define Your Cadence:** While this rollout outlines a standard 8-week launch window, simulations should be rotated quarterly, and assessments should be performed annually to prevent knowledge decay.
3.  **Coordinate with IT early:** Ensure the reporting button (e.g., "Report Phishing" Outlook add-in) is installed and that test domains won't trigger real firewall blocks during weeks 5 and 6.
4.  **Emphasize positive reinforcement:** Never use security simulations as a tool for public shame or direct disciplinary actions on first offenses. Focus on building a community of "Security Champions".
